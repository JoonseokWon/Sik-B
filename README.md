# Sik-B — Food-Fee 정산 PoC

Sik-B는 프로젝트와 저장소 이름이고, Food-Fee는 이 저장소에서 구현한 엔터테인먼트 식비, 정산 관리 기능의 이름입니다.

출퇴근 기록과 일정표로 예상 식사인원을 산정하고, 식비를 정산 대상자에게 배분하며, 예외 건과 승인 이력을 실제 Excel 파일로 남기는 로컬 웹 PoC입니다.

## 해결하려는 문제

엔터테인먼트 현장에서는 연습, 방송, 외부 행사 과정에서 식비가 자주 발생하지만 실제 식사인원과 정산 제외 인원을 사람이 건별로 확인해야 합니다. Sik-B는 다음 과정을 한 화면에서 처리하는 것을 목표로 합니다.

1. 출퇴근 기록과 일정표에서 예상 식사인원을 불러옵니다.
2. 매니저, 스태프, 게스트와 식사 생략, 별도 식사 인원을 정산 대상에서 제외합니다.
3. 실제 식비를 정산 대상자에게 균등 배분합니다.
4. 금액 또는 결제 횟수 기준에 해당하는 건을 승인 흐름으로 분리합니다.
5. 계산 근거와 변경 이력을 `.xlsx` 파일로 출력합니다.

모든 소액 비용을 복잡하게 추적하기보다 일반 건은 자동 처리하고 예외 건만 사람이 확인하도록 설계한 것이 핵심입니다.

## 주요 기능

- 활동 기록, 일정표를 이용한 날짜별 예상 식사인원 산정
- 식비 금액과 정산 대상 인원에 따른 1인 금액 자동 계산
- 식사 생략, 별도 식사, 기타 정산 제외 인원 반영
- 샐러드 주문 등 특이 건의 사유와 실제 식비 직접 수정
- 공통 매출과 개인 매출 구분 및 멤버별 계약 지급률 적용
- 마케팅 사업부 급여를 아이돌별 투입시간 기준으로 배부해 정산금에서 상계
- 로그인 계정별 화면, 입력, 출력, 승인, 로그 열람 권한 분리
- 로그인, 수정, 승인, 출력, 권한 차단 이력을 감사 로그로 기록

## 정산과 승인 흐름

### 식비 배분

예를 들어 실제 식비가 55,000원이고 정산 대상자가 3명이면 1인 금액은 55,000원 ÷ 3명으로 계산합니다. 매니저, 스태프, 게스트는 식사 동석자로 표시될 수 있지만 멤버 정산 대상에서는 제외됩니다.

식사를 하지 않은 멤버는 `식사 생략 인원`, 따로 식사한 멤버는 `별도 식사 인원`에서 선택합니다. 샐러드 주문처럼 일반 배분과 다른 건은 `특이 체크` 후 비고에 사유를 입력하고 실제 식비를 수정할 수 있습니다. 변경 전후와 사유는 감사 로그에 남습니다.

### 승인 기준

다음 중 하나라도 해당하면 자동 처리하지 않고 승인 필요 건으로 분류합니다.

- 1인 금액이 기본 30,000원을 초과한 경우
- 동일 날짜, 동일 시간대에 기본 3건 이상 결제된 경우

금액과 횟수 기준은 화면의 정산 조건에서 변경할 수 있습니다. 동일 시간대는 같은 날짜의 같은 시간 단위로 판정합니다.

승인 상태는 `자동 처리`, `승인 대기`, `승인 완료`, `반려`, `보류`로 관리합니다. 승인 권한이 있는 계정에는 승인 필요 식비 건에 `승인` 버튼이 표시되며, 승인자와 처리 결과가 감사 로그 및 Excel에 반영됩니다.

## 역할별 내부통제

README에는 내부통제 구조만 표시하고, 더미 사번과 비밀번호는 `sample-data/사원정보/`의 계정 Excel에서 별도로 관리합니다.

| 역할 | 입력 | Excel 출력 | 승인 | 감사 로그 | 업무 범위 |
| --- | :---: | :---: | :---: | :---: | --- |
| 정산 담당자 | 가능 | 불가 | 불가 | 열람 불가 | 계정에 배정된 아이돌의 식비 정산과 관련 활동, 일정 관리 |
| 상위 매니저 | 가능 | 가능 | 불가 | 열람 불가 | 정산 데이터 입력, 검토 및 Excel 출력 |
| 인사 마스터 | 가능 | 가능 | 가능 | 열람 가능 | 전체 데이터 관리, 식비 승인, 감사 로그 확인 |

- 정산 담당자는 계정별 담당 아이돌이 고정됩니다. `FIN-1024`는 삼일돌(Samildol), `FIN-1025`는 삼데헌 데이터가 로그인 시 자동 연동됩니다.
- 정산 담당자 화면에는 식비 정산에 필요한 정보만 표시하며 멤버별 지급 정산금액, 매출, 계약, 마케팅 급여는 생성하지 않습니다.
- 상위 매니저에게는 감사 로그 화면을 표시하지 않고, 상위 매니저가 출력한 Excel에서도 감사 로그 섹션을 제외합니다.
- 입력, 출력, 승인 권한을 모두 가진 인사 마스터만 식비 승인 버튼을 사용할 수 있습니다.

## 매출과 계약 계산

- 공통 매출은 공연, 콘서트, 단체 행사처럼 여러 멤버가 함께 만든 매출 풀로 관리합니다.
- 개인 매출은 방송, 라디오, 광고처럼 특정 멤버에게 귀속되는 매출로 관리합니다.
- 멤버별 정산금은 공통, 개인 매출에 계약 지급률을 적용한 뒤 식비와 마케팅 급여 배부액을 공제해 계산합니다.
- 회사 지급률과 멤버별 계약 지급률의 합계가 100%가 되도록 검증합니다.

샘플의 회사, 멤버 지급률과 계약 조건은 계산 및 화면 기능을 검증하기 위한 임의의 더미 가정입니다. 실제 업계 표준, 특정 회사의 계약 관행 또는 법률, 회계 기준을 의미하지 않습니다. 상세 더미 계약서는 `sample-data/아이돌 계약정보/`에서 관리합니다.

## Excel 출력

`Excel 내보내기`는 CSV나 HTML 표가 아닌 실제 `.xlsx` 파일을 브라우저에서 생성합니다. 금액은 숫자 값으로 저장하고 Excel에서 천 단위 구분이 보이도록 서식을 적용합니다.

출력 파일에는 다음 근거가 포함됩니다.

- 정산 요약과 승인 기준
- 매출 항목과 멤버별 정산
- 비용별 식비 배분과 승인 필요 건
- 마케팅 급여 배부
- 활동 기록과 일정표
- 계약서 참조
- 감사 로그(열람 권한이 있는 계정의 출력에 한함)

특이 체크된 식비에 사유가 없으면 Excel 출력을 차단합니다. 식비 배분표에서는 요청에 따라 `내역` 칼럼을 제외합니다.

## 구현 방식

| 구분 | 구현 내용 |
| --- | --- |
| 화면 | HTML과 CSS로 구성한 반응형 로컬 웹 화면 |
| 로직 | 순수 JavaScript로 식비 배분, 매출 정산, 급여 상계, 승인 판정 구현 |
| 데이터 저장 | `localStorage`에 정산 데이터와 감사 로그 저장 |
| 로그인 세션 | `sessionStorage`에 현재 브라우저 세션의 로그인 계정 저장 |
| Excel 출력 | JavaScript에서 Open XML 구성 파일을 ZIP으로 묶어 실제 `.xlsx` 생성 |
| 실행 파일 | C# WinForms 실행 파일이 프로젝트의 `index.html`을 기본 브라우저로 실행 |

별도 백엔드나 데이터베이스 없이 동작하며, `index.html`을 직접 열거나 `Sik-B.exe`를 실행할 수 있습니다.

## 실행 방법

1. 프로젝트 폴더에서 `Sik-B.exe`를 실행합니다.
2. 로그인 화면에서 PoC 계정으로 로그인합니다. 더미 계정은 `sample-data/사원정보/`의 Excel을 확인합니다.
3. 정산 담당자로 로그인하면 계정에 배정된 Samildol 또는 삼데헌 데이터가 자동으로 연동됩니다. 상위 매니저와 인사 마스터는 현재 연동된 아이돌 데이터를 확인합니다.
4. 식비, 활동, 일정, 매출 정보를 확인하거나 권한 범위 안에서 수정합니다.
5. 출력 권한이 있는 계정으로 `Excel 내보내기`를 실행합니다.

## 샘플데이터

- `sample-data/사원정보/`: PoC 사번, 더미 비밀번호, 계정별 권한표
- `sample-data/아이돌 계약정보/`: 아이돌별 더미 계약서와 지급률 근거
- `sample-data/아이돌 출퇴근 정보/`: Samildol, 삼데헌 활동 기록 Excel
- `sample-data/식사 정보/`: Samildol, 삼데헌 식비 결제 샘플 Excel

모든 인물, 그룹, 계약, 급여, 매출, 식비는 기능 검증을 위한 가상 데이터입니다.

## PoC 범위와 한계

- 로그인은 사내 인증이 아닌 클라이언트 측 PoC이며 더미 비밀번호가 코드에 포함돼 있습니다.
- 데이터는 브라우저 저장소에만 보관되므로 사용자, 기기 간 공유, 동시 편집, 중앙 백업을 지원하지 않습니다.
- 권한 검사는 화면과 클라이언트 로직에서 수행되므로 실제 운영 환경의 보안 통제로 사용할 수 없습니다.
- 활동 기록, 일정, 결제 내역은 기능 검증용 더미 데이터로 제공합니다.
- 개인정보 암호화, 서버 감사 로그, 전자결재, ERP, 회계 시스템 연동은 구현 범위에 포함하지 않았습니다.
- 정산과 계약 계산은 기능 검증용이며 실제 지급, 세무, 회계 또는 법률 판단의 근거로 사용할 수 없습니다.
- C# 실행 파일은 웹 화면을 여는 실행 편의 기능이며 별도의 서버나 업무 로직을 포함하지 않습니다.

## 파일 구성

- `index.html`: 화면 구조와 로그인 UI
- `styles.css`: 화면 디자인과 역할별 노출 제어
- `app.js`: 더미 데이터, 권한, 계산, 승인, Excel 입출력 로직
- `launcher/SikBLauncher.cs`: C# 실행 파일 소스
- `Sik-B.exe`: 원클릭 실행 파일
- `_start.bat`: `index.html` 실행용 보조 배치 파일
- `sample-data/`: 사원, 계약, 활동 기록, 식사 샘플데이터
