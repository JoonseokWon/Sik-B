# Sik-B, Food-Fee 정산 PoC

Sik-B는 엔터테인먼트 현장의 식비 배분과 아이돌 정산 업무를 검증하기 위한 웹 PoC입니다. 출퇴근 기록과 일정으로 예상 식사인원을 산정하고, 예외 인원을 제외한 식비 배분, 승인 상태 관리, 감사 로그, `.xlsx` 입력을 한 화면에서 처리합니다. 파일 기반 공용 저장 서버를 실행하면 같은 네트워크의 여러 기기에서 그룹별 데이터를 공유할 수 있습니다.

## 주요 기능

- 출퇴근 기록과 일정에 따른 예상 식사인원 산정
- 식사 생략, 별도 식사, 기타 정산 제외 인원을 반영한 1인 식비 계산
- 샐러드 주문 등 예외 건의 멤버별 식비 지정과 나머지 금액 균등 배분
- 1인 금액과 동일 시간대 결제 횟수에 따른 승인 필요 건 분류
- 공통 매출, 개인 매출, 계약 지급률을 반영한 멤버별 정산
- 마케팅 사업부 급여 중 아이돌별 실제 투입시간에 해당하는 금액만 배부하고 정산금에서 상계
- 계정별 입력, 다른 아이돌 연동, 승인, 감사 로그 열람 권한 분리
- 로그인, 수정, 승인, Import, 권한 차단 이력 기록

## 식비 배분과 승인

실제 식비는 정산 대상 인원에게 균등하게 배분하고 1인 금액은 원 단위로 올림합니다. 매니저, 스태프, 게스트는 식사 동석자로 기록할 수 있지만 멤버 정산에서는 제외됩니다. 식사를 하지 않은 멤버는 식비 배분에서 제외합니다. 별도로 식사한 멤버는 공통 식비 배분에서는 제외하지만 특이 개별 식비 입력 목록에는 유지하며, 금액을 입력하면 해당 별도 식비만 정산합니다.

일반 배분과 다른 건은 `특이 체크` 후 `특이 개별 식비`와 비고를 입력합니다. 개별 금액을 입력한 멤버는 공통 식비 배분에서 제외하고 해당 특이 식비만 부담합니다. 기존 공통 식비는 줄이지 않고 금액을 입력하지 않은 정산 대상 멤버끼리 균등하게 부담합니다. 예를 들어 공통 식비가 90,000원이고 4명 중 한 명이 20,000원짜리 샐러드를 따로 먹었다면 해당 멤버에게 20,000원, 나머지 3명에게는 공통 식비를 각각 30,000원씩 배부합니다. 일반 인원의 1인 금액은 원 단위로 올림합니다.

승인 기준에는 공통 식비 배부액과 특이 개별 식비를 포함한 멤버별 실제 배부액 중 가장 큰 금액을 사용합니다. 공통 식비가 남아 있는데 모든 정산 대상에게 특이 개별 식비를 지정해 일반 배분 대상이 없으면 `보류`로 처리합니다. 기본 식비, 멤버별 특이 개별 식비와 사유의 변경 이력은 감사 로그에 남습니다.

다음 중 하나라도 해당하면 `승인 대기` 상태가 됩니다.

- 1인 금액이 기본 30,000원을 초과한 경우
- 동일 날짜의 같은 시간대에 기본 3건 이상 결제된 경우

예를 들어 12:05와 12:50 결제는 같은 12시 시간대로 계산합니다. 반려된 결제는 동시간대 결제 건수와 멤버 정산에서 제외합니다. 정산 대상자가 0명이면 자동으로 `보류` 상태가 됩니다.

금액과 횟수 기준은 화면에서 변경할 수 있습니다. 일반 건은 `자동 처리`, 특이 체크된 예외 건과 승인 기준에 해당하는 건은 `승인 대기`가 기본입니다. 기존 저장 데이터에서 예외 건에 자동 지정됐던 `보류`도 `승인 대기`로 변환합니다. 인사 마스터가 식비 카드 우측의 빠른 승인 메뉴에서 선택할 수 있는 결과는 `승인 완료`, `반려`, `보류` 세 가지이며, 인사 마스터가 직접 선택한 `보류`는 그대로 유지합니다. 상태 변경자와 변경 전후 상태는 감사 로그에 기록합니다. 식비는 `자동 처리` 또는 `승인 완료` 상태일 때만 멤버 정산에 배부하며, `승인 대기`, `보류`, `반려` 상태에서는 배부하지 않습니다.

## 역할과 권한

입력 범위는 다음과 같습니다.

| 역할 | 식비, 활동, 일정 | 매출, 지급률 | 마케팅 급여 |
| --- | :---: | :---: | :---: |
| 현장 매니저 | 입력 가능 | 입력 불가 | 입력 불가 |
| 상위 매니저 | 입력 가능 | 입력 가능 | 입력 불가 |
| 인사 마스터 | 입력 가능 | 입력 가능 | 입력 가능 |

Import와 내부통제 권한은 다음과 같습니다.

| 역할 | 항목별 `.xlsx Import` | 다른 아이돌 연동 | 승인 | 감사 로그 |
| --- | :---: | :---: | :---: | :---: |
| 현장 매니저 | 불가 | 불가 | 불가 | 열람 불가 |
| 상위 매니저 | 불가 | 가능 | 불가 | 열람 불가 |
| 인사 마스터 | 가능 | 가능 | 가능 | 열람 가능 |

- 현장 매니저는 계정에 배정된 아이돌의 식비, 활동 기록, 일정과 식비 승인 기준을 관리합니다. 멤버별 지급액, 매출, 계약, 마케팅 급여 화면은 표시되지 않습니다. `FIN-1024`는 삼일돌, `FIN-1025`는 삼데헌에 자동 연결됩니다.
- 상위 매니저는 `다른 아이돌 연동`에서 `.xlsx` 파일을 선택해 아이돌 데이터를 가져오고 정산을 검토할 수 있습니다. 마케팅 사업부 급여와 감사 로그는 표시되지 않습니다.
- 인사 마스터는 전체 데이터 열람과 수정, 다른 아이돌 `.xlsx` 연동, 항목별 `.xlsx Import`, 식비 승인, 감사 로그 확인 권한을 가집니다.
- 인사 마스터나 상위 매니저가 연동한 데이터는 계정별 저장값이 아니라 그룹 공용 저장값으로 저장되며, 해당 그룹 현장 매니저가 로그인할 때 같은 데이터를 불러옵니다.

더미 계정은 `sample-data/사원정보/Food-Fee_더미_사원_로그인_계정.xlsx`에서 확인할 수 있습니다.

## 매출과 계약 계산

- 공통 매출은 여러 멤버가 함께 만든 매출 풀로 관리합니다.
- 개인 매출은 방송, 라디오, 광고처럼 특정 멤버에게 귀속되는 매출로 관리합니다.
- 멤버별 지급액은 계약 지급률을 적용한 매출에서 식비와 마케팅 급여 배부액을 공제해 계산합니다.
- 회사 지급률과 멤버별 계약 지급률의 합계가 100%가 아니면 화면에 경고를 표시하고 브라우저 저장을 차단합니다. 수정 중인 값은 현재 화면에 유지되며 합계를 다시 100%로 맞추면 저장됩니다.

## 마케팅 급여 배부

마케팅 직원의 업무시간은 아이돌별 투입시간과 비아이돌 업무시간으로 나눕니다. 아이돌 정산에는 다음 계산식으로 산정한 금액만 상계합니다.

```text
아이돌별 배부액 = 월 급여 × 해당 아이돌 투입시간 ÷ 전체 업무시간
전체 업무시간 = 모든 아이돌 투입시간 합계 + 비아이돌 업무시간
```

비아이돌 업무시간에 해당하는 급여는 `비배부 급여`로 표시하고 어느 아이돌에게도 배부하지 않습니다. 더미데이터의 월 전체 업무시간은 18일 × 점심시간을 제외한 유급 8시간인 144시간으로 설정하며, 직원별 아이돌 투입시간은 116~120시간, 비아이돌 업무시간은 24~28시간입니다. 같은 데이터 버전에서 비아이돌 업무시간 필드만 없으면 144시간에서 아이돌 투입시간 합계를 뺀 시간을 초기값으로 적용합니다. 데이터 버전이 변경되면 기존 브라우저 저장값 대신 최신 더미데이터를 다시 불러옵니다. 이 시간과 급여 데이터도 기능 검증을 위한 가상 값입니다.

샘플 지급률과 계약 조건은 기능 검증을 위한 가상 데이터이며 업계 표준이나 특정 회사의 계약 관행을 의미하지 않습니다.

## `.xlsx Import`

인사 마스터는 Microsoft Excel 통합 문서 형식인 `.xlsx` 파일을 선택해 현재 아이돌의 데이터를 갱신할 수 있습니다.

- 출입 기록: `활동기록` 시트의 회사 입출 기록
- 일정: `일정표` 시트와 `활동기록` 시트의 그룹 캘린더 상태
- 식비 결제: `결제내역`, `Samildol 식사`, `삼데헌 식사` 시트

출입 기록은 날짜와 멤버, 일정은 날짜와 일정명, 식비는 날짜와 결제 시간과 내역을 기준으로 같은 데이터를 교체합니다. 같은 파일을 다시 Import해도 해당 키의 데이터가 중복되지 않습니다. 현재 버전은 `.csv`, `.xls`, Google Sheets 직접 연동을 지원하지 않습니다.

상위 매니저와 인사 마스터는 `다른 아이돌 연동`을 눌러 `활동기록`, `일정표`, `결제내역` 시트가 포함된 `.xlsx` 파일을 선택할 수 있습니다. 멤버 이름으로 삼일돌 또는 삼데헌을 식별하고 해당 아이돌의 출퇴근 기록, 일정, 식비 결제를 한 번에 가져옵니다. 버튼 이름은 현재 아이돌과 관계없이 항상 `다른 아이돌 연동`으로 표시됩니다.

항목별 `.xlsx Import`에서는 파일의 멤버, 아이돌별 식사 시트와 파일명을 이용해 대상 그룹을 먼저 확인합니다. 삼데헌에서 삼일돌 자료를 선택하거나 삼일돌에서 삼데헌 자료를 선택하는 등 현재 그룹과 다른 일정이나 식비 파일이면 해당 연동칸에 `대상 아님`을 표시하고 원본 데이터를 저장하지 않습니다.

선택한 `.xlsx` 파일에서 직접 가져오는 데이터는 출퇴근 기록, 일정, 식비 결제입니다. 멤버 계약, 매출, 계약 지급률, 마케팅 급여는 파일에서 가져오지 않고 앱에 내장된 해당 아이돌의 기능 검증용 더미데이터를 사용합니다.

## 구현 방식과 운영체제

- 화면: HTML, CSS 기반 반응형 웹 UI
- 업무 로직: 순수 JavaScript
- 기본 데이터 저장: 브라우저 `localStorage`, 그룹 공용 키 사용
- 공용 데이터 저장: Python 표준 라이브러리 서버와 `local-data/food-fee-state.json`
- 로그인 세션: 브라우저 `sessionStorage`
- `.xlsx` 처리: JavaScript에서 ZIP과 Open XML을 읽어 Import
- Windows 실행기: C# WinForms 기반 `Sik-B.exe`

업무 화면은 WinForms가 아니라 브라우저에서 실행되는 웹 UI입니다. `Sik-B.exe`는 `index.html`을 기본 브라우저로 여는 Windows 전용 보조 실행기이며 서버나 업무 로직을 포함하지 않습니다. 여러 계정과 기기에서 데이터를 공유하려면 `server.py`를 실행해야 합니다.

- Windows 공용 저장 모드: `_start.bat` 실행
- macOS, Linux 공용 저장 모드: `sh start_server.sh` 또는 `python3 server.py`
- 단일 브라우저 모드: `Sik-B.exe` 또는 `index.html` 실행

macOS와 Linux에서는 WinForms 기반 `.exe`를 실행할 수 없지만 웹 화면과 Python 공용 저장 서버는 사용할 수 있습니다. `.xlsx Import`는 브라우저의 `DecompressionStream` 지원이 필요하며, 지원하지 않는 브라우저에서는 Import 오류를 표시합니다.

## 실행 방법

1. 공용 저장이 필요하면 Windows에서 `_start.bat`, macOS와 Linux에서 `python3 server.py`를 실행합니다. Python 3가 필요하며 브라우저는 자동으로 열리지 않습니다.
2. 서버 창에 표시된 `http://127.0.0.1:8765/`를 같은 기기에서 열거나, `http://호스트의-LAN-IP:8765/`를 같은 네트워크의 다른 기기에서 엽니다.
3. 단일 브라우저에서만 사용할 경우 `Sik-B.exe` 또는 `index.html`을 직접 열 수 있습니다.
4. `sample-data/사원정보/`의 더미 계정으로 로그인합니다.
5. 현장 매니저 계정은 담당 아이돌의 그룹 공용 데이터에 자동 연결됩니다.
6. 상위 매니저와 인사 마스터는 `다른 아이돌 연동`에서 활동 기록 `.xlsx` 파일을 선택해 다른 아이돌 데이터를 가져올 수 있습니다.
7. 인사 마스터는 출입 기록, 일정, 식비 결제의 항목별 `.xlsx Import`를 실행할 수 있습니다.
8. 화면의 저장 상태에서 브라우저 저장 또는 공용 저장 완료 여부를 확인합니다.

- `sample-data/식사 정보/`: 삼일돌, 삼데헌 식비 결제 `.xlsx`

마케팅 직원, 월 급여, 아이돌별 투입시간과 비아이돌 업무시간은 별도 Excel이 아니라 `app.js`에 내장된 더미데이터입니다.

모든 인물, 그룹, 계약, 급여, 매출, 식비는 기능 검증을 위한 가상 데이터입니다.

## PoC 범위와 한계

- 로그인과 권한 검사는 클라이언트에서 수행하며 더미 비밀번호가 코드에 포함돼 있습니다.
- `index.html`을 직접 연 단일 브라우저 모드는 기기 간 공유를 지원하지 않습니다. `server.py` 모드에서는 같은 서버에 접속한 기기끼리 그룹 데이터를 공유합니다.
- 공용 저장 서버는 마지막 저장값 우선 방식이며 동시 수정 충돌 병합, 사용자별 서버 인증, 중앙 백업은 지원하지 않습니다. 신뢰할 수 있는 로컬 네트워크에서만 사용해야 합니다.
- 저장 데이터의 버전이 앱의 데이터 버전과 다르면 마이그레이션하지 않고 최신 더미데이터로 초기화합니다.
- 감사 로그는 브라우저에 최근 80건까지 저장하고 화면에는 최근 12건을 표시합니다.
- 개인정보 암호화, 서버 감사 로그, 전자결재, ERP, 회계 시스템 연동은 구현 범위에 포함하지 않았습니다.
- 정산과 계약 계산 결과는 실제 지급, 세무, 회계, 법률 판단의 근거로 사용할 수 없습니다.

## 파일 구성

- `index.html`: 화면과 로그인 UI
- `styles.css`: 화면 디자인
- `app.js`: 데이터, 권한, 계산, 승인, `.xlsx` 입력 로직
- `server.py`: 그룹 공용 JSON 저장과 정적 파일 제공 서버
- `launcher/SikBLauncher.cs`: Windows 실행기 소스
- `Sik-B.exe`: Windows용 보조 실행기
- `_start.bat`: Windows용 공용 저장 서버 실행 파일
- `start_server.sh`: macOS, Linux용 공용 저장 서버 실행 파일
- `sample-data/`: 사원, 계약, 활동 기록, 식비 샘플데이터
