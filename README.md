# Food-Fee를 계산하는 간단한 툴

엔터산업 정산 과정에서 식비처럼 자주 발생하지만 사람이 일일이 나누기 번거로운 비용을 빠르게 계산하고, 계산 근거를 Excel 파일로 남기는 로컬 웹 툴입니다.

## 실행 방법

1. 프로젝트 폴더에서 `Sik-B.exe`를 더블 클릭합니다.
2. 브라우저가 열리면 `Samildol 더미 연동` 버튼으로 예시 데이터를 불러옵니다.
3. 매출 항목, 멤버 역할, 활동 기록, 일정표, 식비 결제 건을 확인하거나 수정합니다.
4. `Excel 내보내기`를 누르면 정산 근거가 포함된 `.xlsx` 파일이 다운로드됩니다.

별도 설치 없이 `index.html`을 직접 열어도 사용할 수 있습니다.

## 핵심 아이디어

Food-Fee는 모든 비용을 완벽하게 1원 단위로 추적하려는 툴이 아닙니다. 작은 비용 처리에 과도한 인력이 들어가는 문제를 줄이기 위해, 출퇴근 기록과 일정표로 예상 식사인원을 산출하고 예외 건만 승인 흐름으로 분리합니다.

예를 들어 배달 식비 55,000원을 3명이 먹었다면 정산 대상 3명에게 균등 배분합니다. 다만 1인 금액이 30,000원을 넘으면 자동 처리하지 않고 승인 필요 건으로 표시합니다.

## 매출 계산 기준

- 공통 매출: 공연, 콘서트, 단체 행사처럼 여러 멤버가 함께 만든 매출입니다. 멤버별 원금으로 쪼개지 않고 공통 매출 풀로 따로 집계합니다.
- 개인 매출: 방송 출연료, 라디오 DJ, 개인 광고처럼 특정 멤버가 벌어온 매출입니다. 해당 멤버의 개인 매출 원금으로 기록합니다.
- 멤버별 정산금: 공통 매출 풀과 개인 매출 원금에 계약 지급률을 곱한 뒤, 해당 멤버에게 배분된 식비를 공제합니다.
- 계약 지급률은 회사 분배금과 멤버별 분배금의 합계가 100%가 되도록 관리합니다. 현재 더미 기준은 회사 50%, 멤버 합계 50%입니다.
- 아이돌 계약은 균등 조건이 아니라 개인 활동 기여도, 방송/광고 수요, 연습생 생활비 부담, 회사 투자 회수 조건에 따라 지급률이 달라질 수 있다고 가정합니다.
- 매니저, 스태프, 게스트는 식비 제공 또는 고정급 처리 대상으로 보고 정산 대상에서 제외합니다.
- Manager Park는 멤버가 아니라 스태프 더미 데이터입니다. 식비 동석자로는 표시될 수 있지만 멤버별 정산과 계약 지급률 표에서는 제외됩니다.

회사 분배금 50% 가정은 보도된 업계 사례를 기준으로 둔 PoC 기본값입니다. Business Insider는 전직 아이돌 발언을 인용해 행사 섭외비에서 회사가 50%를 가져가고 제작/활동 비용도 회사와 아이돌이 50:50으로 부담하는 경우를 소개했습니다.

참고 출처: Business Insider, `A K-pop idol breaks down the math behind why your favorite star might be broke` (2026-07-08)

## 계약서 참조

앱의 멤버별 정산 표에서 `계약 보기`를 누르면 멤버별 계약 지급률 근거를 확인할 수 있습니다. 더미 계약서 원문은 `contracts/` 폴더에 멤버별 Markdown 파일로 들어 있습니다.

현재 Samildol 더미 계약 지급률:

- 회사: 50%
- Haru: 13%, 개인 활동 기여도 상, 연습생 생활비 본인 부담, 회사 투자 낮음
- Min: 11%, 개인 활동 기여도 상, 연습생 생활비 일부 본인 부담, 회사 투자 중간
- Seo: 10%, 개인 활동 기여도 중상, 연습생 생활비 회사 부담, 회사 투자 높음
- Lia: 9%, 개인 활동 기여도 중, 연습생 생활비 회사 부담, 회사 투자 높음
- Noa: 7%, 개인 활동 기여도 하, 연습생 생활비 회사 부담, 회사 투자 높음

지급률 합계는 회사 50% + 멤버 합계 50% = 100%입니다.

## 활동 기록과 일정표

출퇴근 기록은 회사 입출 기록이 있는 경우 그 데이터를 기준으로 사용할 수 있습니다. 회사로 출근하지 않아도 공연, 방송, 외부 행사처럼 일정표에 참여자로 기록된 경우 활동한 것으로 인정할 수 있습니다.

현재 버전은 실제 API를 연결하지 않고 `Samildol`이라는 가상 그룹의 더미 데이터를 사용합니다.

- 활동 기록: 날짜, 멤버, 출근/대기/외부일정/제외 상태
- 일정표: 날짜, 일정명, 참여 멤버
- 식비 결제 건: 날짜, 내역, 금액, 예상 식사인원, 정산 제외 인원, 승인 상태
- 매출 항목: 날짜, 내역, 공통/개인 구분, 금액, 참여 또는 귀속 멤버
- 멤버 목록: 화면에서 수기 추가하지 않고 회사 내부 그룹 마스터 데이터에서 동기화한다고 가정합니다.

## 승인 기준

- 기본 승인 기준은 1인 30,000원 초과입니다.
- 승인권자는 기본값으로 상위 매니저를 사용합니다.
- 승인 상태는 자동 처리, 승인 대기, 승인 완료, 반려, 보류로 관리합니다.
- 승인 대기/보류/반려 건은 Excel에 별도 섹션으로 정리됩니다.

## Excel 내보내기

내보내기 파일은 실제 `.xlsx` 형식입니다. 금액은 숫자 값으로 저장하며 Excel에서 천 단위 콤마가 보이도록 서식을 적용합니다.

포함되는 근거 자료:

- 정산 요약
- 매출 항목
- 멤버별 정산
- 비용별 분배
- 승인 필요 건
- 활동 기록
- 일정표
- 감사 로그

## 실데이터 연동 방향

현재는 PoC 단계라 로컬 브라우저 저장소와 더미 데이터를 사용합니다. 실제 사내 배포 시에는 다음 구조가 필요합니다.

- 출퇴근 앱: 회사 입출 기록 API 또는 CSV 업로드와 연동
- 캘린더/일정표: 그룹 일정 API 또는 캘린더 export와 연동
- 결제 내역: 법인카드, 배달 플랫폼, 회계 시스템 export와 연동
- 내부 계정: 회사 SSO/사번/권한 시스템과 연동해 입력자, 승인권자, 출력자를 검증
- 권한 관리: 정산 담당자, 매니저, 승인권자 역할 분리
- 보안: 개인정보와 결제 정보는 서버 저장소 또는 사내 보안 저장소에 보관
- 감사 로그: 누가 언제 어떤 데이터를 입력, 수정, 삭제, 승인, 출력했는지 작업자 ID와 권한까지 저장
- 회계 연동: 필요 시 회계팀 양식 또는 ERP 업로드 양식으로 Excel 확장

## 파일 구성

- `Sik-B.exe`: 원클릭 실행 파일
- `_start.bat`: 보조 실행 파일
- `index.html`: 화면 구조
- `styles.css`: 화면 디자인
- `app.js`: 더미 데이터, 계산 로직, Excel 내보내기
- `contracts/`: Samildol 멤버별 더미 계약서
