# Sik-B, Food-Fee 정산 PoC

Sik-B는 엔터테인먼트 현장의 식비 배분과 아이돌 정산 업무를 검증하기 위한 로컬 웹 PoC입니다. 출퇴근 기록과 일정으로 예상 식사인원을 산정하고, 예외 인원을 제외한 식비 배분, 승인, 감사 로그, `.xlsx` 입출력을 한 화면에서 처리합니다.

## 주요 기능

- 출퇴근 기록과 일정에 따른 예상 식사인원 산정
- 식사 생략, 별도 식사, 기타 정산 제외 인원을 반영한 1인 식비 계산
- 샐러드 주문 등 예외 건의 사유 기록과 실제 식비 수정
- 1인 금액과 동일 시간대 결제 횟수에 따른 승인 필요 건 분류
- 공통 매출, 개인 매출, 계약 지급률을 반영한 멤버별 정산
- 마케팅 사업부 급여를 아이돌별 투입시간에 따라 배부하고 정산금에서 상계
- 계정별 입력, `.xlsx Export`, 승인, 감사 로그 열람 권한 분리
- 로그인, 수정, 승인, Export, 권한 차단 이력 기록

## 식비 배분과 승인

실제 식비는 정산 대상 인원에게 균등하게 배분합니다. 매니저, 스태프, 게스트는 식사 동석자로 기록할 수 있지만 멤버 정산에서는 제외됩니다. 식사를 하지 않은 멤버와 별도로 식사한 멤버도 배분 대상에서 제외할 수 있습니다.

일반 배분과 다른 건은 `특이 체크` 후 비고에 사유를 입력하고 실제 식비를 수정합니다. 변경 전후 금액과 사유는 감사 로그에 남습니다.

다음 중 하나라도 해당하면 `승인 대기` 상태가 됩니다.

- 1인 금액이 기본 30,000원을 초과한 경우
- 동일 날짜, 동일 시간대에 기본 3건 이상 결제된 경우

기준값은 화면에서 변경할 수 있습니다. 승인 상태는 `자동 처리`, `승인 대기`, `승인 완료`, `반려`, `보류`로 관리합니다.

## 역할과 권한

| 역할 | 데이터 입력 | `.xlsx Export` | 승인 | 감사 로그 |
| --- | :---: | :---: | :---: | :---: |
| 현장 매니저 | 가능 | 불가 | 불가 | 열람 불가 |
| 상위 매니저 | 가능 | 가능 | 불가 | 열람 불가 |
| 인사 마스터 | 가능 | 가능 | 가능 | 열람 가능 |

- 현장 매니저는 계정에 배정된 아이돌의 식비, 활동, 일정만 관리합니다. `FIN-1024`는 삼일돌, `FIN-1025`는 삼데헌에 자동 연결됩니다.
- 상위 매니저는 아이돌 전환, 정산 검토, `.xlsx Export`를 수행할 수 있습니다. 마케팅 사업부 급여와 감사 로그는 표시되지 않습니다.
- 인사 마스터는 전체 데이터 열람, 아이돌 전환, `.xlsx Import`, 식비 승인, 감사 로그 확인 권한을 가집니다.

더미 계정은 `sample-data/사원정보/Food-Fee_더미_사원_로그인_계정.xlsx`에서 확인할 수 있습니다.

## 매출과 계약 계산

- 공통 매출은 여러 멤버가 함께 만든 매출 풀로 관리합니다.
- 개인 매출은 방송, 라디오, 광고처럼 특정 멤버에게 귀속되는 매출로 관리합니다.
- 멤버별 지급액은 계약 지급률을 적용한 매출에서 식비와 마케팅 급여 배부액을 공제해 계산합니다.
- 회사 지급률과 멤버별 계약 지급률의 합계가 100%인지 검증합니다.

샘플 지급률과 계약 조건은 기능 검증을 위한 가상 데이터이며 업계 표준이나 특정 회사의 계약 관행을 의미하지 않습니다.

## `.xlsx Import`

인사 마스터는 Microsoft Excel 통합 문서 형식인 `.xlsx` 파일을 선택해 현재 아이돌의 데이터를 갱신할 수 있습니다.

- 출입 기록: `활동기록` 시트의 회사 입출 기록
- 일정: `일정표` 시트와 `활동기록` 시트의 그룹 캘린더 상태
- 식비 결제: `결제내역`, `Samildol 식사`, `삼데헌 식사` 시트

같은 키의 데이터는 교체되므로 같은 파일을 다시 Import해도 중복되지 않습니다. 현재 버전은 `.csv`, `.xls`, Google Sheets 직접 연동을 지원하지 않습니다.

## `.xlsx Export`

브라우저에서 Microsoft Excel 통합 문서 형식인 `.xlsx` 파일을 생성합니다. 출력 파일명은 다음 형식입니다.

```text
그룹명_Food-Fee_시작일_종료일.xlsx
```

파일에는 정산 요약, 승인 기준, 매출, 멤버별 정산, 비용별 분배, 승인 필요 건, 활동 기록, 일정, 계약서 참조가 포함됩니다. 인사 마스터의 파일에는 마케팅 급여 배부와 감사 로그도 포함됩니다. 상위 매니저의 파일에서는 두 항목을 제외합니다.

금액은 문자열이 아닌 숫자 값으로 저장하고 천 단위 구분 서식을 적용합니다. 특이 체크된 식비에 사유가 없으면 Export를 차단합니다.

## 구현 방식과 운영체제

- 화면: HTML, CSS 기반 반응형 웹 UI
- 업무 로직: 순수 JavaScript
- 데이터 저장: 브라우저 `localStorage`
- 로그인 세션: 브라우저 `sessionStorage`
- `.xlsx` 처리: JavaScript에서 ZIP과 Open XML을 읽고 생성
- Windows 실행기: C# WinForms 기반 `Sik-B.exe`

업무 화면은 WinForms가 아니라 브라우저에서 실행되는 웹 UI입니다. `Sik-B.exe`는 `index.html`을 기본 브라우저로 여는 Windows 전용 보조 실행기이며 서버나 업무 로직을 포함하지 않습니다.

- Windows: `Sik-B.exe` 또는 `index.html` 실행
- macOS, Linux: `index.html`을 Chrome, Edge, Safari 등 최신 브라우저에서 실행

macOS와 Linux에서는 WinForms 기반 `.exe`를 실행할 수 없지만 웹 화면과 `.xlsx` 입출력 기능은 브라우저에서 사용할 수 있습니다.

## 실행 방법

1. Windows에서는 `Sik-B.exe`를 실행하거나 `index.html`을 엽니다. macOS와 Linux에서는 `index.html`을 엽니다.
2. `sample-data/사원정보/`의 더미 계정으로 로그인합니다.
3. 현장 매니저 계정은 담당 아이돌 데이터에 자동 연결됩니다.
4. 인사 마스터는 출입 기록, 일정, 식비 결제의 `.xlsx Import`를 실행할 수 있습니다.
5. 권한 범위에서 데이터를 확인하고 수정합니다.
6. 상위 매니저와 인사 마스터는 `.xlsx Export`를 실행할 수 있습니다.

## 샘플데이터

- `sample-data/사원정보/`: 더미 로그인 계정과 권한
- `sample-data/아이돌 계약정보/삼일돌/`: 삼일돌 멤버별 더미 계약서
- `sample-data/아이돌 계약정보/삼데헌/`: 삼데헌 멤버별 더미 계약서
- `sample-data/아이돌 출퇴근 정보/`: 삼일돌, 삼데헌 활동 기록 `.xlsx`
- `sample-data/식사 정보/`: 삼일돌, 삼데헌 식비 결제 `.xlsx`

모든 인물, 그룹, 계약, 급여, 매출, 식비는 기능 검증을 위한 가상 데이터입니다.

## PoC 범위와 한계

- 로그인과 권한 검사는 클라이언트에서 수행하며 더미 비밀번호가 코드에 포함돼 있습니다.
- 데이터는 브라우저에 저장되므로 기기 간 공유, 동시 편집, 중앙 백업을 지원하지 않습니다.
- 개인정보 암호화, 서버 감사 로그, 전자결재, ERP, 회계 시스템 연동은 구현 범위에 포함하지 않았습니다.
- 정산과 계약 계산 결과는 실제 지급, 세무, 회계, 법률 판단의 근거로 사용할 수 없습니다.

## 파일 구성

- `index.html`: 화면과 로그인 UI
- `styles.css`: 화면 디자인
- `app.js`: 데이터, 권한, 계산, 승인, `.xlsx` 입출력 로직
- `launcher/SikBLauncher.cs`: Windows 실행기 소스
- `Sik-B.exe`: Windows용 보조 실행기
- `_start.bat`: Windows용 브라우저 실행 배치 파일
- `sample-data/`: 사원, 계약, 활동 기록, 식비 샘플데이터
